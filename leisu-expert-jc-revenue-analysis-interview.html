一、总览：专家竞彩收入业务要回答的 8 个核心问题

核心逻辑：雷速体育的商业化重点不只是流量变现，而是让赛事流量在专家模块里形成可信内容消费、付费/跳转转化和持续复购。

建议的整体指标树：
专家竞彩收入 = 专家模块访问用户 × 方案点击率 × 付费/跳转转化率 × 客单价 × 复购频次
专家模块访问用户 = 赛事流量 × 专家入口曝光率 × 入口点击率
复购收入 = 历史转化用户 × 回访率 × 再次购买/转化率 × 客单价
长期价值 = 专家命中体验 × 内容可信度 × 用户复购 × 用户留存

1. 用户为什么会进入专家模块？

核心判断 找到用户从“看比赛/比分”切到“看专家推荐”的真实触发点。

分析目标
  • 识别专家模块访问的主要入口和触发场景。
  • 判断用户是被赛事不确定性、专家品牌、入口曝光还是活动刺激驱动。
  • 找到专家入口最应该出现的位置和时机。
关键数据
  • 入口来源：比分页、赛程页、赛事情报页、热门赛事页、推送、搜索。
  • 赛前行为：查看赔率、阵容、伤停、历史交锋、情报、指数变化。
  • 专家行为：入口曝光、点击、列表停留、专家关注、方案详情点击。
  • 转化结果：购买/跳转、客单价、复购、退款/投诉、评价。

实操分析方案

  1. 做入口归因：统计专家模块访问前最后一个页面和最近 30 分钟的赛事行为。
  2. 做场景分组：赛前 24 小时、赛前 2 小时、赛中、赛后分别看专家入口点击率和转化率。
  3. 做意图行为识别：用户是否查看了赔率变化、伤停、阵容、情报、历史交锋等高决策意图内容。
  4. 做入口效率对比：比较比分页、赛程页、情报页、推送、搜索带来的转化质量和复购质量。

指标和 SQL 思路

-- 伪 SQL：专家模块访问前 30 分钟行为归因
WITH expert_visit AS (
  SELECT user_id, event_id, visit_time, source_page
  FROM dwd_expert_module_visit
  WHERE dt BETWEEN '2026-06-01' AND '2026-06-30'
),
pre_behavior AS (
  SELECT
    v.user_id,
    v.event_id,
    v.source_page,
    b.event_name,
    b.page_name,
    b.event_time
  FROM expert_visit v
  JOIN dwd_user_event_behavior b
    ON v.user_id = b.user_id
   AND b.event_time BETWEEN v.visit_time - INTERVAL '30' MINUTE AND v.visit_time
)
SELECT source_page, event_name, COUNT(DISTINCT user_id) AS users
FROM pre_behavior
GROUP BY source_page, event_name
ORDER BY users DESC;

业务动作

  • 如果情报页转化质量最高，把专家入口和相关专家推荐嵌入情报消费后链路。
  • 如果赛前 2 小时转化最高，重点配置赛前推送和热门赛事专家位。
  • 如果搜索带来的用户复购高，优化专家和赛事关键词承接。
  • 如果入口曝光高但点击低，优化入口文案、专家标签和赛事相关性。

面试对答稿：

“我不会把专家模块当成孤立页面分析，而会先看用户为什么从赛事内容进入专家模块。雷速体育的用户通常先有赛事需求，比如看比分、看赛程、看赔率和伤停，再产生对专家判断的需求。”

“我的分析会从专家访问前 30 分钟行为入手，区分比分页、赛事情报页、热门赛事页、推送和搜索这些入口的转化效率。最终要回答的是：专家入口应该在什么赛事、什么时间、什么用户行为之后出现，才能带来高质量转化。”

2. 哪些赛事和用户场景最能驱动专家内容消费？

核心判断 专家收入强依赖赛事热度、赛前决策时点和用户关注强度。

分析目标
  • 找到高转化赛事类型和高价值时间窗口。
  • 判断热门赛事和长尾赛事分别适合什么专家供给策略。
  • 识别哪些赛事情报因素会拉动专家内容消费。
关键数据
  • 赛事属性：联赛、球队、热度、开赛时间、强弱对比、是否焦点战。
  • 用户场景：赛前、赛中、赛后、周末、深夜、杯赛、联赛。
  • 内容消费：专家列表浏览、方案详情、专家关注、情报页深度。
  • 结果指标：购买/跳转转化、客单价、复购、次日/7日回访。

实操分析方案

  1. 按赛事热度分层：S 级焦点赛、A 级热门赛、普通赛、长尾赛，分别看专家消费和收入。
  2. 按时间窗口拆解：赛前 24 小时、赛前 6 小时、赛前 2 小时、赛中、赛后复盘。
  3. 按赛事信息复杂度拆解：赔率波动大、阵容不确定、伤停多、强弱悬殊、历史交锋明显等。
  4. 做场景贡献分析：收入变化到底来自赛事流量增加，还是专家点击率/转化率提升。

赛事场景矩阵

场景 用户心理 分析重点 业务策略
焦点赛赛前 2 小时 决策意图强，信息需求集中 专家入口点击率、方案购买率、专家排名 配置头部专家、突出近期表现和核心观点
赔率波动明显赛事 用户需要解释变化原因 赔率页到专家详情页转化 联动指数解读和专家观点
冷门联赛长尾赛事 信息稀缺，专家价值可能更高 低流量高转化赛事识别 少量高质量专家覆盖，控制供给成本
赛中比分胶着 即时判断和情绪波动强 赛中访问、实时内容点击、跳转转化 强化实时观点和赛中提醒

业务动作

  • 热门赛事优先保障专家数量、头部专家露出和赛前内容更新。
  • 冷门赛事重点筛选高转化、高复购专家，不盲目铺量。
  • 在赔率波动、阵容公布、伤停变化后触发专家观点推荐。
  • 用赛事热度预测模型提前安排专家供给和运营位。

面试对答稿：

“专家竞彩收入不是均匀发生的，它强依赖赛事场景。我会把赛事按热度、联赛、开赛时间和信息复杂度分层，看哪些场景最能驱动专家内容消费。”

“比如焦点赛赛前 2 小时可能流量和决策意图都很强，适合头部专家露出；冷门联赛流量小，但如果信息稀缺，专家内容的转化率反而可能高。这个分析的目的，是让专家供给和运营位跟赛事价值匹配，而不是平均分配资源。”

3. 专家内容转化漏斗哪里流失最大？

核心判断 收入问题要先拆漏斗，找出是流量不足、点击不足、信任不足，还是支付/跳转不足。

实操分析方案

  1. 建立专家转化漏斗：入口曝光 → 入口点击 → 专家列表浏览 → 方案详情 → 购买/跳转 → 后续复购。
  2. 按来源、赛事、专家、用户分层拆漏斗，定位掉点最大的环节。
  3. 对详情页做行为分析：停留、滑动、历史战绩点击、评论查看、价格区域曝光、退出路径。
  4. 分析支付/跳转后链路：支付成功率、跳转成功率、回流率、购买后再次访问。
入口点击率 专家入口点击 UV / 专家入口曝光 UV，用于判断入口位置和文案是否有效。
列表到详情率 方案详情 UV / 专家列表 UV，用于判断专家排序、标签和标题吸引力。
详情转化率 购买或跳转 UV / 方案详情 UV，用于判断内容信任和价格阻力。
复购率 周期内二次购买/转化用户 / 首次购买/转化用户，用于判断长期价值。

指标和 SQL 思路

-- 伪 SQL：专家漏斗按入口来源拆解
SELECT
  source_page,
  COUNT(DISTINCT CASE WHEN event_name = 'expert_entry_expose' THEN user_id END) AS expose_uv,
  COUNT(DISTINCT CASE WHEN event_name = 'expert_entry_click' THEN user_id END) AS click_uv,
  COUNT(DISTINCT CASE WHEN event_name = 'expert_list_view' THEN user_id END) AS list_uv,
  COUNT(DISTINCT CASE WHEN event_name = 'expert_plan_detail_view' THEN user_id END) AS detail_uv,
  COUNT(DISTINCT CASE WHEN event_name IN ('plan_pay_success','jc_jump_success') THEN user_id END) AS convert_uv
FROM dwd_expert_funnel_event
WHERE dt BETWEEN '2026-06-01' AND '2026-06-30'
GROUP BY source_page;

业务动作

  • 入口点击低：优化入口位置、赛事情境文案和专家标签。
  • 列表到详情低：优化专家排序、标题、命中展示、价格和赛事相关性。
  • 详情转化低：增强专家可信信息、近期表现、观点摘要和购买理由。
  • 复购低：重点看方案体验、命中后承接、未命中后的解释和专家关注链路。

面试对答稿：

“如果专家收入下降，我第一步不会直接猜专家质量差，而是先拆完整漏斗。入口曝光、入口点击、列表到详情、详情到购买/跳转、后续复购，每一步下降对应的原因完全不同。”

“比如曝光足够但点击低，可能是入口位置和赛事相关性问题；详情页访问多但转化低，可能是价格、信任或专家表达问题；首次转化高但复购低，则说明短期刺激有效但内容体验没有建立长期信任。”

4. 哪些用户最可能产生竞彩相关转化？

核心判断 专家模块不能对所有用户平均商业化，要识别高潜转化和高复购用户。

分析目标
  • 建立专家竞彩转化用户分层。
  • 识别高转化、高复购、高客单人群。
  • 指导专家推荐、运营触达、价格和活动策略。
关键变量
  • 赛事行为：活跃天数、关注球队/联赛、赛前访问、赔率/情报查看。
  • 专家行为：专家列表访问、详情页深度、关注专家、收藏方案。
  • 交易行为：历史购买/跳转、客单价、复购间隔、退款/投诉。
  • 价格行为：优惠点击、低价方案偏好、活动期转化。

用户分层示例

用户类型 数据特征 策略
高频赛事用户 多联赛高频看赛，赛前访问多，关注球队多 推荐赛事相关专家，重点提高入口点击
竞彩兴趣用户 高频查看赔率、指数、情报和专家详情 强化专家观点、近期表现和组合推荐
专家跟随用户 关注固定专家，复购集中在少数专家 做专家更新提醒、订阅和复购承接
价格敏感用户 低价方案和活动转化高，原价复购低 控制补贴，关注 LTV 和复购质量
低信任用户 浏览详情多但转化低，反复看评价和战绩 增强可信背书、历史样本和观点摘要

模型思路

专家转化倾向分 =
  0.25 × 近7天赛前访问天数
+ 0.20 × 赔率/情报页查看次数
+ 0.20 × 专家详情页访问深度
+ 0.15 × 专家关注/收藏行为
+ 0.10 × 热门赛事关注强度
+ 0.10 × 历史转化或跳转行为

评估：
- Top 10% 人群转化率 lift
- 触达组 vs 未触达组增量转化
- 首次转化后的 7/30 日复购率
- 客单价和投诉/退款率

业务动作

  • 高频赛事用户：在其关注赛事赛前强化专家入口。
  • 竞彩兴趣用户：推荐高相关专家和多维情报解读。
  • 专家跟随用户：推专家更新提醒和复购链路。
  • 价格敏感用户：用低门槛方案试用，但用 LTV 控制补贴。

面试对答稿：

“我会把专家收入增长先变成人群经营问题。不是所有比分用户都有专家转化意图，真正高潜的人通常有赛前访问、赔率情报查看、专家详情浏览、关注专家等行为。”

“我会基于这些行为做转化倾向分层，同时关注复购质量。比如价格敏感用户可能转化率高，但原价复购低；专家跟随用户可能规模不大，但 LTV 很高。策略上要对不同人群做差异化推荐和触达。”

5. 专家方案的质量如何影响复购和信任？

核心判断 专家模块的长期收入不是靠一次点击，而是靠用户对专家内容的持续信任。

实操分析方案

  1. 建立专家质量指标：近期表现、稳定性、观点完整度、更新及时性、用户评价、退款/投诉。
  2. 比较专家质量指标和复购率、关注率、客单价、流失率之间的关系。
  3. 分析命中体验：命中、未命中、连续命中、连续未命中后用户行为变化。
  4. 区分短期高转化专家和长期高复购专家，避免排序只追短期收入。
专家转化能力 专家曝光到购买/跳转转化率，反映短期变现能力。
专家复购能力 购买/跳转过该专家的用户在 7/30 天内再次转化比例。
专家稳定性 近期表现的波动程度、连续未命中次数、内容更新稳定性。
信任健康度 关注率、正向评价、退款/投诉、详情页反复查看后的转化。

分析重点

  • 不要只按短期 GMV 排专家，要同时看复购和投诉退款。
  • 命中体验对复购有滞后影响，建议看 T+1、T+7、T+30 行为。
  • 观点解释质量可能影响未命中后的留存，不能只看结果命中。
  • 专家内容稳定更新会影响用户关注和回访。

业务动作

  • 专家排序加入复购率、关注率和信任健康度，不只按收入排序。
  • 对高复购专家增加曝光，对高投诉高收入专家降权观察。
  • 对连续未命中专家做流量控制和内容质量复盘。
  • 引导专家提供清晰逻辑和赛后复盘，降低用户信任损耗。

面试对答稿：

“专家方案质量对收入的影响分短期和长期。短期看点击和转化，长期看复购、关注、评价、退款投诉和流失。如果只按短期收入排序，可能会牺牲用户信任。”

“我会给专家建立质量分层：转化能力、复购能力、稳定性和信任健康度。最终专家排序和运营不能只看 GMV，而要平衡短期收入和长期复购，这样专家模块才是可持续生意。”

6. 用户为什么复购、沉默或流失？

核心判断 首次转化只是起点，复购才决定专家收入的长期质量。

实操分析方案

  1. 建立首购 cohort：按首次购买/跳转日期跟踪 D1、D7、D30 回访和复购。
  2. 分析首购后体验：是否查看赛果、是否关注专家、是否继续浏览同类赛事、是否评价/投诉。
  3. 拆用户状态：复购用户、只回访不复购、沉默用户、流失用户。
  4. 识别流失原因：近期未命中、专家断更、赛事兴趣下降、价格敏感、入口触达不足。
用户状态 数据表现 运营动作
高复购用户 固定专家关注，高频购买/跳转，多赛事回访 专家订阅、更新提醒、组合推荐
回访不复购用户 仍看赛事和专家详情，但不再转化 增强信任信息、优化价格和观点摘要
沉默用户 首购后 7/14 天未访问专家模块 基于关注赛事或专家做召回
流失用户 赛事活跃也下降，专家和比分均不活跃 先做赛事内容激活，再承接专家推荐

指标和 SQL 思路

-- 伪 SQL：首购 cohort 复购分析
WITH first_convert AS (
  SELECT user_id, MIN(convert_time) AS first_time
  FROM dwd_expert_convert
  GROUP BY user_id
),
repeat_convert AS (
  SELECT
    f.user_id,
    f.first_time,
    COUNT(CASE WHEN c.convert_time BETWEEN f.first_time AND f.first_time + INTERVAL '7' DAY THEN 1 END) AS d7_convert_cnt,
    COUNT(CASE WHEN c.convert_time BETWEEN f.first_time AND f.first_time + INTERVAL '30' DAY THEN 1 END) AS d30_convert_cnt
  FROM first_convert f
  LEFT JOIN dwd_expert_convert c
    ON f.user_id = c.user_id
   AND c.convert_time > f.first_time
  GROUP BY f.user_id, f.first_time
)
SELECT
  DATE(first_time) AS cohort_date,
  COUNT(*) AS first_users,
  AVG(CASE WHEN d7_convert_cnt > 0 THEN 1 ELSE 0 END) AS d7_repeat_rate,
  AVG(CASE WHEN d30_convert_cnt > 0 THEN 1 ELSE 0 END) AS d30_repeat_rate
FROM repeat_convert
GROUP BY DATE(first_time);

业务动作

  • 首购后引导关注专家和关注赛事，形成下次回访理由。
  • 命中后及时承接同专家/同赛事推荐，提高复购。
  • 未命中后提供解释和复盘内容，减少信任断裂。
  • 沉默用户用关注赛事开赛提醒召回，而不是泛促销。

面试对答稿：

“我会把专家业务的用户生命周期分成首次转化、赛果体验、复购、沉默和流失。首次购买不是终点，关键是用户是否愿意再次相信专家。”

“分析上我会做首购 cohort，看 D7、D30 复购，并结合命中体验、专家关注、赛事回访来解释原因。业务动作上，命中后要承接复购，未命中后要减少信任损耗，沉默用户要用关注赛事和专家更新召回。”

7. 专家供给、赛事热度和收入之间如何匹配？

核心判断 专家供给要跟赛事需求匹配，热门赛事保收入，长尾赛事找效率。

实操分析方案

  1. 构建赛事热度预测：基于历史流量、联赛、球队、比赛时间、用户关注数预测赛事需求。
  2. 统计每场赛事专家供给：专家数、方案数、头部专家覆盖、更新时间、价格分布。
  3. 建立供需匹配矩阵：高热度高供给、高热度低供给、低热度高供给、低热度低供给。
  4. 分析供给效率：每个赛事、联赛、专家的曝光转化、收入、复购、投诉退款。
供需类型 问题 策略
高热度高供给 竞争强，排序影响大 优化专家排序和差异化标签
高热度低供给 收入机会损失 提前邀约专家，补充头部和垂类专家
低热度高供给 曝光不足，供给浪费 减少运营位，控制方案供给
低热度高转化 小流量高价值 识别垂类专家和稳定用户群，精细化运营

关键指标

  • 赛事专家覆盖率：有专家方案的赛事数 / 可运营赛事数。
  • 高热赛事供给充足率：高热赛事中达到专家数量和质量阈值的比例。
  • 单位曝光收入：赛事专家收入 / 专家入口曝光。
  • 供给浪费率：低曝光、低转化方案占比。
  • 专家供给及时率：开赛前指定时间内完成方案发布的比例。

业务动作

  • 提前预测高热赛事，保障头部专家和垂类专家覆盖。
  • 高热赛事用多专家、多观点满足不同用户；长尾赛事用少量高质量专家。
  • 对低热低转化赛事减少供给和运营资源。
  • 建立专家供给排班和赛事运营看板。

面试对答稿：

“专家业务的收入上限不仅取决于用户需求，也取决于专家供给是否匹配赛事热度。高热赛事如果专家供给不足，会损失收入；低热赛事如果供给过多，会浪费资源。”

“我会做一个赛事热度和专家供给的矩阵，把赛事分成高热高供给、高热低供给、低热高供给和低热高转化。这样业务上就能提前安排专家供给，把资源投到最有收入和复购潜力的赛事上。”

8. 专家竞彩收入增长由什么驱动，是否可持续？

核心判断 收入增长要拆成流量、转化、客单、复购和信任健康，而不是只看 GMV。

收入指标树

专家竞彩收入 = 新增转化收入 + 复购收入 + 活动收入
新增转化收入 = 专家新转化用户 × 首购客单价
复购收入 = 历史转化用户 × 回访率 × 再次购买/跳转率 × 复购客单价
收入变化 = 赛事流量变化 + 专家入口效率变化 + 方案详情转化变化 + 客单价变化 + 复购变化

实操分析方案

  1. 做收入拆解：按新增、复购、活动、专家、赛事、入口来源拆收入贡献。
  2. 做增长质量分析：收入增长是否伴随复购提升、投诉退款下降、专家关注提升。
  3. 做活动评估：活动是否带来新增增量，还是提前透支高潜用户。
  4. 做专家结构分析：收入是否过度依赖少数专家或少数赛事。
分析问题 对应方法 判断标准
收入上涨是不是健康？ 拆新增、复购、客单、退款/投诉、专家关注 复购和信任指标同步提升，才是高质量增长
增长是不是靠赛事红利？ 剔除重大赛事后看常规赛事收入 如果只靠大赛，常规增长能力不足
是否依赖少数专家？ 看 Top 专家收入占比和复购贡献 过度集中会带来供给风险
活动是否透支未来？ 比较活动用户后续复购和自然用户 LTV 低价转化但复购差，说明增长质量偏低

业务动作

  • 每周收入复盘同时汇报新增、复购、客单、专家结构和信任健康度。
  • 降低对单一赛事和单一专家的收入依赖。
  • 活动预算优先给高复购潜力人群，而不是只追短期转化。
  • 把专家排序和运营目标从短期收入升级为收入 + 复购 + 信任健康。

面试对答稿：

“专家竞彩收入增长要拆开看，不能只看 GMV。收入可能来自赛事流量变大、入口点击提升、详情转化提升、客单价提升，也可能来自复购提升。不同驱动代表完全不同的业务质量。”

“我会特别看增长是否可持续：复购率有没有提升，专家关注有没有提升，投诉退款有没有恶化，收入是否过度依赖少数专家或重大赛事。如果只是活动刺激带来短期转化，但后续复购差，那就不是健康增长。”

30 秒版本

“我会把雷速体育专家竞彩收入拆成一条完整链路：赛事流量进入专家入口，用户消费专家内容，产生购买或跳转转化，再通过专家质量和赛果体验形成复购。分析上不是只看收入，而是同时看入口效率、转化漏斗、用户分层、专家质量、赛事供需匹配和复购健康度。”

“如果我来负责雷速体育专家竞彩收入分析，我会先搭指标树：专家竞彩收入等于专家模块访问用户乘以方案点击率、付费/跳转转化率、客单价和复购频次。专家访问又来自赛事流量、专家入口曝光和入口点击。这样收入问题就能被拆成流量、入口、内容、转化和复购几个层级。”

“实操上我会做三类分析。第一是场景分析，找到哪些赛事、哪些时间窗口、哪些用户行为最能驱动专家内容消费。第二是漏斗和人群分析，定位曝光、点击、详情、购买/跳转和复购哪里流失，并识别高潜转化用户。第三是专家和供给分析，评估专家质量、复购能力、信任健康度，以及专家供给是否匹配赛事热度。最后业务动作会落到入口优化、专家排序、赛事供给、复购召回和活动效率上。”

高频追问与回答

追问 回答要点
专家收入下降，你怎么分析？ 先拆收入指标树：赛事流量、入口曝光、入口点击、详情转化、客单价、复购。再按赛事、入口、专家、人群分层定位贡献最大的下滑环节。
转化率下降可能是什么原因？ 可能是入口相关性下降、专家列表排序变差、详情页信任不足、价格阻力提升、专家近期表现变差，或跳转/支付链路异常。要按漏斗定位。
如何提升专家模块复购？ 首购后引导关注专家和赛事，命中后承接同专家推荐，未命中后提供复盘解释，对沉默用户用关注赛事和专家更新召回。
如何评价一个专家好不好？ 不能只看收入，要看曝光转化、复购率、关注率、近期稳定性、用户评价、退款投诉和对长期留存的影响。
热门赛事专家供给怎么做？ 提前预测赛事热度，高热赛事保障头部和垂类专家覆盖，低热赛事控制供给。用赛事热度 × 专家供给矩阵判断资源是否错配。
如何判断增长是否可持续？ 看复购率、专家关注、用户留存、退款投诉、收入集中度和活动用户后续 LTV。如果收入上涨但复购下降或投诉上升，增长质量偏低。

分析师落地节奏

  1. 第一周：统一专家收入口径，搭建收入指标树和完整漏斗看板。
  2. 第二周：完成赛事场景和入口来源分析，识别最高价值赛事窗口。
  3. 第三周：完成人群分层和转化倾向模型，输出触达策略。
  4. 第四周：完成专家质量和复购分析，调整专家排序和运营规则。
  5. 第五周后：沉淀专家收入周报和异常诊断机制，持续监控收入、复购和信任健康度。
