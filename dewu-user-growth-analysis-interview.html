01

得物化业务模型：潮流消费、交易信任与供给流动性

面试中先建立的业务判断

得物的增长不是普通货架电商的“流量 × 转化”。它要同时经营买家需求、SKU/尺码/卖家供给、实时价格与平台信任。用户完成支付只是交易中段；发货、鉴别、履约时效、售后与下一次关注/搜索，才共同决定长期价值。

需求与内容

种草 → 搜索 → 关注

内容、热点发售、品牌/球鞋潮流和价格提醒，决定用户有没有明确购买意图。

供给与价格

SKU/尺码 → 卖家 → 报价

可售率、尺码覆盖、报价深度与价格波动，决定用户想买时能否以可接受价格成交。

信任与履约

支付 → 发货 → 鉴别 → 签收

鉴别通过、履约时效、异常处理与售后体验，决定信任、退款、复购和口碑。

得物化交易漏斗

内容/外部投放触达 → 搜索/浏览商品 → 查看 SKU 与尺码
→ 价格比较/关注/加购 → 提交订单/支付 → 卖家发货
→ 平台鉴别/履约 → 签收/售后 → 关注、复购与再次交易

因此，支付转化下降不能只从落地页找原因；也要看目标 SKU 的可售率、价格偏离、卖家发货能力、鉴别异常和预计到货时效。

与普通电商不同的核心指标

模块得物化指标增长含义
供给匹配目标 SKU 可售率、尺码覆盖率、报价深度用户有意图时是否“买得到”
价格发现价格波动、报价竞争度、关注后成交率价格提醒和补贴是否改变决策
信任履约卖家发货时效、鉴别通过率、预计到货偏差、售后率首单转化与长期复购的护栏
内容需求内容到商品点击、搜索转化、发售节点转化种草流量能否变成交易意图
得物化面试对答稿：“我会把得物理解为以潮流消费需求为起点、以交易信任为核心、由供给流动性决定效率的平台型电商。增长不是只把更多用户导到下单页，而是要让高意图用户在合适的 SKU、尺码和价格下完成可信交易。所以我会同时看内容/搜索需求、SKU 可售率和报价、支付转化，以及发货、鉴别、到货、售后对复购的影响。补贴和投放的最终判断，也必须回到增量贡献毛利与信任履约护栏。”
02

核心指标树：从预算到长期价值

增量贡献毛利 = 增量首单毛利 + 增量复购毛利 − 渠道成本 − 补贴成本 − 履约/退款损失
获客效率曝光 → 点击 → 注册 → 激活 → 首单
CAC、CPA、CVR、首单成本
补贴效率券曝光 → 领取 → 使用 → 增量下单
券增量 ROI、敏感度、蚕食率
用户质量D1/D7/D30 留存、复购、GMV、毛利
LTV/CAC、退款率、客诉率
渠道 ROI = (增量贡献毛利 − 渠道成本) / 渠道成本 券增量 ROI = (实验组贡献毛利 − 对照组贡献毛利) / 补贴成本 LTV/CAC = 预计生命周期贡献毛利 / 获客成本
Q1

怎样判断增长健康，而非被补贴“买出来”？

业务问题

新增和首单上涨时，如何判断是有效增长，还是低质量流量、价格套利或对自然订单的替代？

实操分析方案

1建立双层北极星短期看新客首单与 CAC；长期看 D30 复购贡献毛利、LTV/CAC、退款和客诉。
2做新客 Cohort按渠道、首单品类、券类型和价格带比较 D1/D7/D30、复购、毛利和退款。
3做增量拆分把自然增长、投放增量、补贴增量分开；重大决策优先用随机对照。

示意图：不是看谁首单转化最高，而是看谁在 D30 后仍贡献毛利。

面试对答稿：“我不会把新增或首单当成唯一成功标准。我会按获客渠道、首购品类和券类型做 Cohort，把 CAC、D30 贡献毛利、复购与退款并列。若某渠道首单成本低但 D30 退款高、复购低，我会识别为低质量增长。投放和补贴的扩量依据是实验增量与 LTV/CAC，而不是表面转化。”
Q2

投放渠道预算如何分配，何时加码、何时止损？

业务问题

多个渠道都声称“转化好”时，如何在归因偏差和边际效应递减下分配预算？

示意数据：扩量看边际 ROI，不能只看平均 ROI。

实操分析方案

1统一归因底表按渠道、计划、素材、日期沉淀消耗、归因新增、站内订单、退款和贡献毛利。
2估计边际 ROI按预算分位或预算阶梯实验，测量“多花 1 元带来多少增量贡献毛利”。
3设置预算规则边际 ROI 高于阈值且质量护栏稳定则加码；连续低于阈值则降档或暂停。
面试对答稿：“我会把预算优化做成闭环：先统一平台消耗、MMP 归因和站内订单/退款；再按渠道和预算阶梯评估边际 ROI；最后把预算向高增量、高质量人群迁移。我的止损条件不会只看 CPA，而是边际贡献毛利低于门槛，或 D7 留存、退款率出现恶化。”
Q3

优惠券怎么发，才能提高增量而不是补贴自然订单？

业务问题

同一张券的核销率很高，是否就值得加大补贴？如何找到券额、门槛和人群的最优组合？

实操分析方案

1先定义补贴人群区分新客未购、加购未付、沉默召回、高频自然购买者和价格敏感用户，重点防后两类蚕食。
2设计实验矩阵以券额 × 门槛 × 人群 × 触达时机为因子，不以领取者为样本，而以被随机分配者为样本。
3按增量毛利结算计算增量下单、GMV、补贴、履约和退款，输出券增量 ROI 与敏感度曲线。

示意图：券额通常存在转化提升与毛利损失的交点。

样例 SQL：券组与对照组的意向分析
WITH cohort AS (
  SELECT user_id, experiment_group, assigned_at
  FROM growth_experiment_assignment
  WHERE experiment_name = 'new_user_coupon_v1'
)
SELECT experiment_group,
       COUNT(*) AS users,
       AVG(CASE WHEN o.order_id IS NOT NULL THEN 1 ELSE 0 END) AS order_rate_7d,
       AVG(COALESCE(o.contribution_margin, 0)) AS margin_per_assigned_user
FROM cohort c
LEFT JOIN order_user_7d o ON c.user_id = o.user_id
GROUP BY 1;
面试对答稿：“券核销率只说明领取者会不会用，不能说明补贴是不是增量。我会把对象随机分到不同券额和对照组，用意向分析计算每个被分配用户带来的 7 天增量订单和贡献毛利。高频自然购买者重点看蚕食率；加购未付用户可能用低券额和限时触达就够。最终用增量毛利/补贴成本选策略。”
Q4

首单转化下降，如何从漏斗定位到能落地的动作？

业务问题

注册后不下单，究竟是流量意图、商品供给、价格心智、信任机制，还是支付履约环节的问题？

示意漏斗：先找绝对损失和转化变化共同异常的节点。

实操分析方案

1统一事件链路商品曝光、点击、搜索、加购、提交订单、支付成功、退款；补充版本、货品、价格带和端信息。
2做贡献度分解同时看步骤覆盖人数、环比变化、渠道构成和异常集中的人群，而非只盯最低转化率。
3形成问题—假设—实验例如加购到支付流失，先排查券可用性、库存、价格与支付错误，再决定产品或运营实验。
诊断纪律：漏斗告诉你“哪里发生”，日志、客服反馈、价格/库存维度与实验才帮助确认“为什么发生”。不要把相关性直接说成因果。
面试对答稿：“我会先校验漏斗的去重规则，再用绝对损失和转化变化做贡献度定位。以加购到支付为例，我会继续按新老客、价格带、券状态、库存和端版本拆分，并结合错误码和客服原因验证。最后才转成可实验动作，例如优化券可用性提示，同时守住退款和客诉护栏。”
Q5

如何定义高质量新客，把获客与留存经营连起来？

业务问题

不同渠道首单成本接近，为什么后续复购与毛利差异很大？

示意图：首单周表现接近，但留存曲线会在 D7 后明显分化。

实操分析方案

1建立首购生命周期表固定首单日期，追踪 D1/D7/D30 活跃、复购、GMV、毛利、退款和优惠依赖度。
2定位留存关键行为检验首日关注、搜索、收藏、浏览内容/鉴别信息等行为与后续留存的关系。
3区分运营策略低意图人群优化首购激活；高价值沉默用户做商品/内容/价格提醒，避免无差别补券。
面试对答稿：“我会把渠道评估窗口从首单延长到 30 天，并把贡献毛利、复购和退款并列。若两个渠道 CPA 相同，但 A 渠道 D30 毛利更高，我会提高它的可接受 CAC 上限。首日关键行为可以用于人群运营线索，但其因果效果仍要通过实验确认。”
Q6

多源数据口径冲突时，怎样做归因与增长诊断？

业务问题

广告平台、MMP、站内订单和财务的转化/成本不一致时，怎样避免经营决策被多算或少算带偏？

数据源价值典型偏差治理动作
媒体平台消耗、素材、点击、平台回传多平台重复认领用于平台内优化，不直接作为全局增量结论
MMP / 归因层统一设备/渠道归因跨端、隐私限制、归因窗口差异沉淀 attribution_version 和窗口规则
站内行为与订单注册、首单、退款、复购埋点丢失、状态回流延迟订单事实表与状态机校验
财务成本真实消耗、补贴、履约、退款结算周期不同日估算与月结回溯两套口径
面试对答稿：“我不会强行把所有数据源做成一个数字，而是先定义每张表的决策用途：媒体用于优化，MMP 用于跨渠道统一，站内订单和财务用于经营结算。重要预算决策以真实订单、退款和实验增量为准，并将归因窗口、去重规则与数据延迟写入指标说明。”
Q7

增长实验怎样设计，才能既快又可信？

业务问题

面对发券、页面、触达和投放人群等提案，怎样避免“谁声音大就上线谁”？

实验 SOP

1预先写清假设与阈值例如“门槛券提升加购未付新客的 7 日增量贡献毛利”，并写明最小可检测效果与停止规则。
2随机、互斥、分层用户级随机；处理多券/多触达互斥；按新老客、渠道或品类分层。
3主指标与护栏并行主指标优先用增量贡献毛利/用户；护栏看退款、毛利率、留存、客诉和自然订单蚕食。

实验设计卡

实验：加购未付新客的限时门槛券 实验组：随机 50%，展示限时券与可用商品提示 对照组：随机 50%，维持原路径 主指标：7 日增量贡献毛利 / 分配用户 护栏：退款率、毛利率、D7 留存、客诉率 决策：显著提升主指标且护栏不劣化，才扩大
面试对答稿：“我会把实验当作决策基础设施，而不是上线后的复盘工具。每次先确定假设、随机单元、主指标和停止规则。券类实验尤其要监控自然订单蚕食、退款与后续留存。结束时除了 p 值，我还会说明绝对增量、适用人群和是否值得规模化。”
Q8

如何把行业与竞品信号转成可验证的增长策略？

业务问题

竞品活动、品类热点或消费季节变化出现时，分析师怎样提供业务建议，而不是只做资讯搬运？

实操分析方案

1建立外部事件日历消费节点、品类发售、竞品活动、价格变化与内容热点，对齐站内流量、搜索、转化和退款趋势。
2提出机制假设例如竞品补贴导致某价格带比价流失；先区分时间相关与可验证机制。
3小范围验证再扩张在特定人群、品类或地域测试策略，复核边际 ROI 与长期质量后再调整预算与券。

经营节奏

日

消耗、异常 CVR、库存/支付/券故障预警。

周

边际 ROI、漏斗诊断、实验读数、预算调整。

月

Cohort、回收周期、品类结构和策略复盘。

面试对答稿：“我把行业与竞品信息当作假设来源，而不是直接结论。观察到品类转化下滑时，我会把外部活动、价格和供给事件对齐到站内搜索、加购、支付和退款，确认影响人群后再小范围验证。最终输出的是对哪类用户、在哪个节点、用什么预算或券策略、改善什么指标。”
09

高频追问与 90 秒自我陈述

ROI 很高，为什么还不加预算？

先确认是平均还是边际 ROI，以及是否是增量。高 ROI 若只来自高意图人群，扩量很快会饱和；用预算阶梯和 D30 贡献毛利决策。

券核销率低，怎么做？

先拆券曝光、领取、可用商品、门槛理解和支付环节。核销低不等于加大券额，应用人群与触达时机实验判断。

没有实验条件，怎么评估？

可用地理分组、分批上线、倾向得分匹配或双重差分，但需披露混杂因素；重大预算仍争取随机对照。

第一天发现转化下降，怎么处理？

先排数据/埋点，再确认是否集中在渠道、版本、地区、品类、库存、支付或券；实时止损与根因分析并行。

如何和投放、产品、运营合作？

用同一张经营卡对齐问题、证据、可控变量、预期增量、成本、护栏和决策日期。分析师要让团队用同一套事实行动。

你的核心原则？

相关性用于定位，实验用于决策；短期转化用于监控，长期贡献毛利用于经营。每次建议都说清口径、置信度、风险和验证路径。

90 秒自我陈述模板

“我理解这个岗位的关键，不是单纯把投放数据做得更漂亮，而是把获客、补贴、转化与用户长期价值串成经营闭环。我的做法通常有三步：第一，和业务统一增量贡献毛利、CAC、LTV/CAC 口径，建立从渠道到订单再到留存的指标链；第二，通过漏斗、分群和 Cohort 定位瓶颈，判断问题是在流量质量、商品/价格、券策略还是产品链路；第三，对投放和补贴用实验评估增量，并以退款、毛利和留存做护栏。这样我输出的不只是数据结论，而是可执行的预算、人群、券和产品策略，以及明确的验证方案。”
